Retrieval configuration guide

Tune which repository evidence CodeAtlas retrieves before asking the LLM.

Workflow: select a repository → Load → make a small JSON change → Save → test the same question above. Changes apply to future retrieval immediately; reindexing is not required.

What each field controls and improves

stopwords
Common words removed before matching. Keep domain terms such as “checkout” or “authentication”.Benefit: cleaner search signals and fewer irrelevant matches.
synonyms
Expands a question term into vocabulary used by the codebase.Benefit: finds the right code even when users and developers use different words.
keyword_boosts
Score multipliers for especially important search terms. Start around 1.5–2.0.Benefit: lifts business-critical features above generic matches.
preferred_components
Prioritizes relevant classes, screens, services, repositories, or modules by name.Benefit: anchors answers to known architectural entry points.
preferred_methods
Prioritizes relevant fully qualified methods, such as AuthRepository.login.Benefit: brings exact implementation paths into the evidence.
node_limit
Maximum graph nodes added to answer context. Higher can add coverage and noise.Benefit: balances answer breadth against relevance and latency.
relation_limit
Maximum relationships between selected graph nodes.Benefit: reveals call and dependency flows without overwhelming the answer.
excerpt_nodes
How many top nodes receive real source-code excerpts.Benefit: grounds more answer claims in verifiable code.
excerpt_max_lines
Maximum source lines attached for each excerpt.Benefit: captures enough surrounding logic while controlling prompt size.
excerpt_max_chars
Hard character cap for each source excerpt.Benefit: keeps prompt size and response time predictable.
allow_shared_fallback
Compatibility field. Use the repository’s Shared LLM toggle for the enforced privacy policy.Benefit: the central toggle keeps privacy behavior explicit and consistent.

Concise real-world examples

Users say “sign in”; code says “auth”

{
  "synonyms": {
    "signin": ["login", "auth"],
    "login": ["signin", "authentication"]
  },
  "keyword_boosts": {
    "auth": 1.8
  }
}

Keep checkout answers focused

{
  "preferred_components": [
    "CheckoutViewModel",
    "PaymentRepository"
  ],
  "preferred_methods": [
    "PaymentRepository.submitPayment"
  ],
  "node_limit": 20,
  "excerpt_nodes": 8
}

Keep the existing JSON structure and data types. Start with synonyms or one boost, compare answer evidence, and increase limits only when important context is missing.
